Turnip

Turnip is a hybrid mobile app built with Ionic2.

It manages food recipes, prints out the ingredients lists, suggests recipes based on local products in season.

Development

I found myself in need of organizing my food recipes, as they were scattered over multiple text files, notes and image files.

As I want to access them when cooking, obviously, but also need access to the list of ingredients when shopping, I created a mobile app to install on iPad and mobile phone.

The recipes are entered on a Wordpress site, accessible to the app through the Wordpress REST API.

The home screen shows recipes based on seasonal ingredients, and user preferences. (e.g. vegetarian, Asian...).

The recipes on the home screen can be filtered by clicking the filter button and selecting a predefined filter.

The search screen allows for detailed search by ingredient, course type, or free text.

On the recipe screen, the export to export to shopping list button exports all ingredients to a shopping list. This list can be edited, and items can be deleted.
A checkbox marks purchased items in strike trough.

Experience with Ionic2

Building with Ionic does speed up the development process. I was able build a mobile app rapidly, having only knowledge of html, css and Javascript. Performance is very good, virtually indistinguishable from native.

Javascript issues can easily be solved by using the debugger in the browser. The emulator can now be used with live reload, so that even scripts that access native functionality can be changed and tested on the fly.

Platform specific issues do pop up. For example the splash screen on android is buggy, and doesn't work out of the box. The rendering in the emulator is different from the rendering of the app installed on the device. Overriding the default CSS styles requires a good understanding of the Ionic platform.
In order to create a finished application, calculating 20% overhead to solve these types of issues seems mandatory.

In a production environment however, these concerns can be delegated to key people in the team with in-depth knowledge of Ionic or the mobile platform. This is obviously not the case in native app development, where every developer needs thorough knowledge of the platform in order to be able to code.
The framework is ideal for rapid prototyping, as it allows to make changes easily, and deployment is effortless.
