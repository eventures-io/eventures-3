About

When I started out in software development, most of the challenge in producing software was technical. Especially so in web development.

These days, the technology has matured, and the challenge has shifted from producing working software to creating user-friendly applications.

This requires a human centered approach, very different from the traditional software development cycle and methodologies.
The question has become:

“Are you going to push features or create an experience“

In order to create a good user experience it is necessary to start from of a contextual view of the users, the problem at hand, and the business. Understand the business goals and business drivers. (the why of the business). Understand how the product can fulfill users' needs. Distill engaging user journeys from this information.

My task as user experience designer is to be able to abstract away the business goals, and map them to goals and tasks that are relevant to the user.

“Because users don’t care about your solution, they care about their problem”

This means a project is successful if:

  • The user can accomplish his task (Business goals are met)
  • The user can do so easily, with minimal friction (User friendliness goal met)
  • The is process enjoyable for the user (A good user experience)
